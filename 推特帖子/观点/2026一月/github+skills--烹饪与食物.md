skills 是一份份食物，但是如何去烹饪出好吃的食物的思路，却是决定每个人差距的必要因素。 

我很久之前就习惯用AI 搜索 github 上面的高 star 工具，然后 AI 进行优化，做成更适合自己用的工具或者脚本。

skills 的出现其实并没有改变原理，只是让所有人能更简单地运行别人打包好的操作。 

我举两个我做知识库的案例： 我之前把哥飞老师的所有公众号文章用 github 上高 star 工具批量下载 html，然后再让 gpt 帮我找了个批量 html to pdf 工具和 pdf 合并工具，然后再把合并后的 pdf 上传到 notebooklm 做成自己的知识库。 

我还把涛哥几十个付费会员的直播回放，用 github 上面的工具批量下载下来，然后批量把 mp4转成 mp3，再用其他工具批量合并成小于100m 的，最后上传到 notebooklm 形成知识库。 

为什么我要让 AI 用 github 上面的高 star 工具，而不是从零开发？ github 上面的开源代码是无数优秀程序员的心血结果，经过了时间的验证。比起 AI 的不确定性，我更想要开源工具的稳定性和确定性，它能帮我节约大量重复造轮子的时间。 

上面这两个案例当然也可以做成 skills，让每个人都能用上，但是，这两个即将诞生的 skills 只是一份食物，如何去烹饪，烹饪的思路，是值得每个人去细细研究和思考的。
