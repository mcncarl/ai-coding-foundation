## 背景
测试新的 /summary skill 格式，去掉 YAML frontmatter 和重复的 H1 标题。

## 核心内容
- 格式调整：无元数据块，直接从内容开始
- 标题优化：文件名即为主题，内容从 H2 开始

## 解决方案

修改后的笔记格式：
```markdown
## 背景
<简述对话背景和问题>

## 核心内容
<主要讨论内容>

## 解决方案
<如果有，列出解决方案>

## 关键要点
- 要点1
- 要点2

## 相关
- 项目路径: <当前工作目录>
- 相关技术/工具
```

## 关键要点
- 无 YAML frontmatter（tags、date 等）
- 无 H1 标题（避免与文件名重复）
- 保持简洁，直接进入主题

## 相关
- 项目路径: D:\
- 相关技术: Claude Code CLI, MCP, Obsidian
