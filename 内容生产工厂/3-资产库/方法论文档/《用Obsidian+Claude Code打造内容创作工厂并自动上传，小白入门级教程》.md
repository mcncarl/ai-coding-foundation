# 一、Obsidian到底是什么
**Obsidian是一款本地化的强大的笔记软件**，可以记录你的所有内容和数据。

# 二、Obsidian有哪些好处
**1. 数据安全性强**  
你的所有素材库、爆款选题、数据复盘都存在自己的电脑里，不用担心云端服务商倒闭或数据泄露。这对内容创作者来说特别重要——**你的选题库就是你的核心资产**。
**2. 高度可定制**  
通过插件，你可以把Obsidian改造成任何你想要的样子。想要AI辅助创作？**装一个Copilot插件**。想要通过思维导图的方式梳理项目？**装一个Ecalidraw插件**。想要记录每次笔记的更改，想要更安全地储存内容？**装一个Git插件**。
**3. 离线可用**  
不需要网络也能工作，相比于飞书的云文档真的要方便很多，在飞机上也可以用obsidian继续创作，nice！

# 三、Obisidian安装教程
## 第一步：下载Obsidian

1. 打开浏览器，访问 https://obsidian.md/
2. 点击页面上的"Download"按钮
3. 选择你的操作系统版本：
    - **Windows用户**：选择 Windows installer (.exe)
    - **Mac用户**：选择 macOS (.dmg)
    - **Linux用户**：选择对应的Linux版本
4. 点击下载，等待文件下载完成

## 第二步：安装Obsidian

**Windows用户：**
1. 找到下载好的 Obsidian 安装文件（.exe）
2. 双击打开，会自动启动安装程序
3. 按照提示点击"Next"，选择安装位置（默认即可）
4. 等待安装完成，自动打开Obsidian

**Mac用户：**
1. 找到下载好的 .dmg 文件，双击打开
2. 将Obsidian图标拖到Applications文件夹
3. 打开Applications，找到Obsidian，双击启动

**安装完成后，你会看到一个欢迎界面。**

## 第三步：创建你的第一个Vault（仓库）

Vault是Obsidian的核心概念——它就是一个文件夹，里面存放你所有的笔记（可以创建多个滴~）
1. 打开Obsidian后，会看到"Create new vault"的选项
2. 输入你的Vault名称，比如"我的内容创作库"
3. 选择存储位置（建议放在一个容易找到的地方，比如桌面或文档文件夹）
4. 点击"Create"，Obsidian会自动创建这个文件夹并打开![](assets/《用Obsidian+Claude%20Code打造内容创作工厂并自动上传，小白入门级教程》/file-20260211173812389.png)

打开Vault后，你会看到这样的界面布局：
**上方：**
- **文件列表**：显示你所有的笔记文件和文件夹

# 五、如何把Claude Code对话的精华内容沉淀到Obsidian

这是我自己每天都在用的实战流程，把和Claude Code的精华对话内容通过自定义的/sunmary命令直接保存到obsidian的“Claude Code对话精华内容”文件夹里面，然后把这些内容作为我的选题来源之一。具体的步骤如下：
![](assets/《用Obsidian+Claude%20Code打造内容创作工厂并自动上传，小白入门级教程》/file-20260211173812368.png)

## 第一步：先建一个“精华接收文件夹”

在你的 Obsidian Vault 里先建一个专用目录，比如：

`E:/obsidian/Claude Code对话精华内容`

为什么要单独建目录？  
后面每次 `/summary` 的输出都会落在这里，后续做复盘、做选题、做二次创作会非常丝滑。

## 第二步：创建 summary 技能

在本地创建这个文件（也可以让Claude辅助创作）：

`C:/Users/HP/.claude/skills/summary/SKILL.md`

内容结构建议这样写（可直接参考我这套）：

```markdown
---
name: summary
description: 将当前对话的精华内容提取并保存到 Obsidian。触发词：/summary、保存对话、导出精华、沉淀到 Obsidian。
---

# 目标
将当前对话提炼为结构化笔记并保存到 Obsidian。

## 提炼标准
- 保留：问题与解决方案、可复用代码/命令、关键决策原因、新知识点、踩坑教训
- 过滤：寒暄、临时日志、无效尝试

## 输出结构
## 背景
## 核心内容
## 解决方案
## 关键要点
## 相关
```

可以自己增添很多细节，比如我的这版增加了：
1. 不要加 YAML frontmatter 到最终笔记正文里；  
2. 不要用 H1，正文直接从 H2 开始；  
3. 文件名避免非法字符（`/ \ : * ? < > |`）。

## 第三步：加一个 `/summary` 命令

直接告诉AI：

```markdown
我希望我在Claude Code 对话界面新增一个/summary命令，每次发出命令都能执行将Claude Code精华内容直接保存到obsidian（调用sunmary skill）。
```


# 六、怎么把Obsidian里面的长文章上传到X

这个skill，之前@wshuyi Wang Shuyi大佬分享过，但是对于windows用户来说，有很多坑要踩，我至少优化了一个周，解决了被检测到自动化浏览器无法登陆X账号、文章图片上传错位、解析特殊符号错误等等问题，才有了这一版本。

其实这个让AI自动化操控浏览器的解决办法也是Claude Opus4.6给我的解决方案，用cookie来自动注入，从而实现X账号的登录。这个方式除了上传文章到X，理论上任何自动操作浏览器的方法都能用到

我实测已经能用一行命令直接上传Obsidian的文章到X，也把/summary的skill和这个上传文章的skill开源到了github，大家可以clone下来玩玩：https://github.com/mcncarl/yichen-skills![](assets/《用Obsidian+Claude%20Code打造内容创作工厂并自动上传，小白入门级教程》/file-20260211173929372.png)
