## 核心原则

### 正确分工

| 角色 | 负责内容 | 不负责内容 |
|-----|---------|-----------|
| **人类** | 创意、拍板、审核 | 执行、整理、生成 |
| **AI** | 执行、整理、生成 | 创意、决策、判断 |

**关键：**
- 人类提供想法和方向
- AI负责展开和执行
- 人类最终审核和优化

## 对话技巧

### 技巧1：分步提问，不要一次性要求太多

**❌ 错误示例：**
```
帮我写一篇关于AI的推特帖子，要有吸引人的开头，
包含3个案例，有金句，结构清晰，字数300-400字，
还要生成配图文案。
```

**✅ 正确示例：**
```
第1轮：我想写一篇推特帖子，主题是AI工具如何提升创作效率。
       请帮我列出3个可以展开的角度。

第2轮：我选择第2个角度。请帮我生成大纲。

第3轮：大纲很好。请按照这个大纲生成完整内容，
       字数300-400字。
```

**为什么分步更好？**
- AI每次只聚焦一个任务，质量更高
- 人类可以在过程中调整方向
- 避免一次性输出太多需要大幅修改

---

### 技巧2：提供具体例子，不要抽象描述

**❌ 错误示例：**
```
写得有真人感一点。
```

**✅ 正确示例：**
```
删除这些AI痕迹词汇：
- "在当今时代"
- "综上所述"
- "值得一提的是"

改用短句和口语化表达。
```

---

### 技巧3：明确约束条件

**❌ 错误示例：**
```
写一篇推特帖子。
```

**✅ 正确示例：**
```
写一篇推特帖子：
- 字数：300-400字
- 结构：问题-观点-案例×3-强调
- 开头：用问题式开头
- 案例：需要具体的人物和数据
- 避免：AI痕迹词汇
```

---

### 技巧4：让AI解释思路，不要直接要答案

**❌ 错误示例：**
```
直接告诉我这篇文章为什么数据好。
```

**✅ 正确示例：**
```
请分析这篇文章的成功要素，从以下维度：
1. 开头是什么类型？为什么有效？
2. 内容结构是什么？逻辑如何？
3. 有什么金句？为什么印象深刻？
4. 案例选择有什么特点？
```

**为什么要让AI解释？**
- 帮助人类理解成功要素
- 提取可复用的模式
- 建立更深层的认知

```

- 目标受众：[如：AI创业者、自媒体新手]
- 核心观点：[一句话总结]
- 案例素材：[列出1-3个案例]

**内容要求：**
1. 开头：[从爆款开头库选择的类型，如"问题式"]
2. 结构：[从爆款结构模板选择，如"问题-观点-案例×3-强调"]
3. 字数：300-400字
4. 案例：具体的人物、数据、场景
5. 避免：AI痕迹词汇（"在当今时代""综上所述"等）

**风格参考：**
[可选：粘贴一篇你写过的高质量推文]

请生成初稿。
```

### 模板2：优化现有内容

```
请优化这篇推文：

[粘贴原文]

**优化方向：**
1. 开头：前2-3句更吸引人
2. 结构：段落是否需要调整？
3. 真人感：删除AI痕迹，改用口语
4. 具体性：抽象描述改为具体案例/数据
```

### 模板3：拆解爆款推文

```
请帮我拆解这篇爆款推文的成功要素：

[粘贴推文内容]

**数据：**
- 曝光：XXw
- 互动：XX（点赞+评论+转发）

**分析维度：**
1. **开头分析**：前2-3句是什么类型？为什么有效？可复用的句式是什么？
2. **结构分析**：整体结构是什么？逻辑如何展开？
3. **金句提取**：有哪些印象深刻的句子？为什么有效？
4. **案例分析**：案例选择有什么特点？是否具体？
5. **成功要素排序**：按权重排列影响数据的因素（开头/结构/话题/时机等）

请按照以上维度输出分析结果。
```

## 常见错误

### 错误1：完全依赖AI，不做人工审核

**问题：**
- AI生成的内容可能有事实错误
- AI可能使用陈词滥调
- AI缺乏真人感

**解决：**
- 必须人工审核
- 检查5项质量清单
- 删除AI痕迹词汇

---

### 错误2：提示词太模糊

**问题：**
```
写一篇好的推文。
```

**解决：**
- 明确主题、字数、结构、风格
- 提供具体例子
- 列出约束条件

---

### 错误3：一次性要求太多

**问题：**
```
写一篇推文，还要配图文案、标题、摘要、SEO关键词...
```

**解决：**
- 分步提问
- 每次聚焦一个任务
- 人类在过程中调整方向

---

### 错误4：不保存有效的提示词

**问题：**
- 每次都重新写提示词
- 忘记之前有效的提问方式

**解决：**
- 把有效的提示词保存到这个文件
- 建立自己的提示词库
- 不断优化和积累

---

### 错误5：忽略AI的"幻觉"

**问题：**
- AI可能编造数据
- AI可能编造案例
- AI可能编造引用

**解决：**
- 所有数据必须人工核实
- 所有案例必须确认真实性
- 不确定的内容不要使用

## AI工具推荐

### Claude（推荐）

**优势：**
- 长上下文（200k tokens）
- 理解能力强
- 可以读取整个Obsidian知识库

**适用场景：**
- 复盘分析
- 拆解爆款
- 生成长内容
- 知识库管理

---

### ChatGPT

**优势：**
- 响应速度快
- 生成质量稳定
- 多模态（可以生成图片）

**适用场景：**
- 快速生成初稿
- 优化润色
- 标题生成

---

### Deepseek

**优势：**
- 免费
- 超长上下文（1M tokens）
- 中文友好

**适用场景：**
- 预算有限时
- 需要超长上下文时
- 中文内容创作

## 进阶技巧

### 技巧1：建立对话记录库

**操作：**
- 把有价值的对话保存到"1-素材输入/对话记录/"
- 标注对话类型（如"拆解分析""选题澄清"）
- 方便以后参考和复用

---

### 技巧2：让AI学习你的风格

**操作：**
```
这是我写的5篇高质量推文：

[粘贴5篇推文]

请总结我的写作风格特点（开头/结构/用词/节奏）。

然后按照这个风格，帮我写一篇新推文，主题是：[新主题]
```

---

### 技巧3：用AI做竞品分析

**操作：**
```
这是3篇同类型的爆款推文：

推文1：[粘贴]
推文2：[粘贴]
推文3：[粘贴]

请分析它们的共性：
1. 开头有什么共性？
2. 结构有什么共性？
3. 案例选择有什么共性？
4. 金句有什么共性？

然后告诉我，如果我要写同类型内容，应该注意什么？
```

---

### 技巧4：用AI做数据预测

**操作：**
```
这是我过去10篇推文的数据：

[粘贴标题和数据]

请分析：
1. 哪种类型的内容数据更好？
2. 哪种开头句式效果更好？
3. 发布时间对数据有影响吗？

基于这些分析，我下篇推文应该：[AI给出建议]
```
